## 🎯 系统架构

```
用户消息
   ↓
主代理（我）
   ├─ QUICK → 立即回复（问候/状态查询）
   ├─ NORMAL → 正常处理（普通对话）
   └─ TASK → spawn 子代理 → 后台执行 → 完成后汇报
```

## 🔧 使用方式

### 方式 1：手动触发（当前）

当你说"执行 XXX 任务"时，我会：

1. **判断任务类型**
   - 搜索/调研 → 子代理
   - 文件操作 → 子代理
   - 简单对话 → 立即回复

2. **创建子代理**
   ```javascript
   sessions_spawn({
     task: "你的任务内容",
     label: "task-worker",
     cleanup: "delete",
     timeoutSeconds: 300
   })
   ```

3. **立即回复你**
   > "收到！正在后台处理，完成后告诉你结果～"

4. **子代理执行**
   - 独立会话运行
   - 结果写入 `reports/` 目录
   - 完成后飞书通知你

### 方式 2：自动触发（待实现）

修改飞书插件 handler，自动分类消息：
```typescript
// extensions/feishu/src/handler.ts
```

## 📊 任务分类规则

### 立即回复（QUICK）
- ✅ 问候：你好、hello、在吗
- ✅ 状态：报告状态、当前模型
- ✅ 短消息：<10 字
- ✅ 确认：是否、能不能

### 子代理处理（TASK）
- ✅ 搜索调研：搜索 XXX、分析 XXX
- ✅ 文件操作：创建、写入、修改
- ✅ 复杂任务：实施、执行、完成
- ✅ 下载配置：下载、安装、配置

### 正常处理（NORMAL）
- 其他普通对话

## 🔒 安全限制

### 子代理权限
| 权限 | 状态 | 说明 |
|------|------|------|
| 读取文件 | ✅ 允许 | 工作区内 |
| 写入文件 | ✅ 允许 | 仅 reports/ |
| 网络访问 | ✅ 允许 | 搜索/查询 |
| Shell 命令 | ❌ 禁止 | 安全考虑 |
| 敏感文件 | ❌ 禁止 | 加密/凭证等 |

### 资源限制
- **超时：** 5 分钟
- **并发：** 最多 2 个子代理同时运行
- **内存：** 继承主代理限制
