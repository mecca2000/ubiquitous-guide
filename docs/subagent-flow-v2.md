# 子代理处理流程 v2.0

**实施时间：** 2026-03-03 23:30  
**核心原则：** 子代理应上尽上，对话永远优先

---

## 🎯 新工作流程

```
收到消息
   ↓
【第 1 步】立即回复（<3 秒）
"收到！正在处理..."
   ↓
【第 2 步】分类判断
├─ QUICK → 直接回复（你好/状态）
├─ TASK → spawn 子代理 + 立即返回
└─ NORMAL → 判断是否>30 秒
              ├─ 是 → spawn 子代理
              └─ 否 → 主代理处理
   ↓
【第 3 步】处理消息队列
- 检查是否有新消息
- 优先处理 HIGH 优先级（对话）
- 子代理后台执行
   ↓
【第 4 步】中途汇报（>30 秒的任务）
"任务进行中，已完成 X%..."
   ↓
【第 5 步】子代理完成
- 读取结果
- 整合汇报
```

## 📋 子代理工作范围（应上尽上）

### ✅ 必须 spawn 子代理的任务

| 类型 | 触发词 | 示例 |
|------|--------|------|
| 🔍 **搜索调研** | 搜索、查找、调研、探索 | "搜索 moltbook" |
| 📊 **数据分析** | 分析、研究、对比 | "分析 AI 趋势" |
| 📝 **文档生成** | 创建、写入、生成、报告 | "创建系统报告" |
| 📥 **下载安装** | 下载、安装、配置 | "安装 XXX 技能" |
| 🎓 **学习探索** | 学习、探索、发现、扫描 | "探索新技能" |
| 📁 **文件操作** | 修改、删除、整理 | "整理文件" |
| ⏱️ **周期任务** | 心跳、检查、监控 | HEARTBEAT 任务 |

### ✅ 主代理直接处理的任务

| 类型 | 示例 |
|------|------|
| 💬 **问候对话** | "你好"、"在吗"、"hello" |
| ⚡ **状态查询** | "报告状态"、"当前模型" |
| 📢 **结果汇报** | 整合子代理结果后汇报 |
| ❓ **决策判断** | 需要人类确认的事项 |

### openclaw.json - 消息队列
```json
"messages": {
  "queue": {
    "type": "memory",
    "maxSize": 1000,
    "priority": {
      "HIGH": ["对话", "状态", "查询", "你好", "在吗"],
      "NORMAL": ["任务", "搜索", "创建", "分析"],
      "LOW": ["后台", "日志", "监控"]
    }
  }
}
```

## 📊 对比改进

### 改进前（❌ 错误）
```
23:01:43 收到"搜索技能"
   ↓
主代理执行搜索（1 分钟）
   ↓
23:02:43 搜索完成 → 汇报
   ↓
23:02:00 的"汇报状态"消息被延迟 ❌
```

### 改进后（✅ 正确）
```
23:01:43 收到"搜索技能"
   ↓
立即回复："收到！正在后台搜索～"（<3 秒）
   ↓
spawn worker-task 执行（后台）
   ↓
立即返回处理消息队列
   ↓
23:01:50 看到"汇报状态"
   ↓
立即回复："我在线！搜索中，已发现 X 个..."（<3 秒）
   ↓
继续对话...
   ↓
23:02:43 子代理完成 → 汇总汇报
```
