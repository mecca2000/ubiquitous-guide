# 子代理系统优化方案 v3.0

**优化原则：** 精简、按需、避免冗余  
**时间：** 2026-03-03 20:45

---

## 🎯 问题反思

### 原设计（4 个子代理）的问题
| 问题 | 说明 |
|------|------|
| ❌ **过度分工** | 4 个代理职责有重叠 |
| ❌ **资源浪费** | 每个都要独立内存/CPU |
| ❌ **调度复杂** | 需要判断任务类型再分配 |
| ❌ **可能堵塞** | 多个代理并发可能占满资源 |

---

## ✅ 优化方案：2 个子代理

### 新架构

```
主代理（我）
├─ 对话交互（立即回复）
└─ 任务调度
    ├─ worker-task（通用任务执行者）
    └─ worker-cron（定时任务执行者）
```

---

## 📋 子代理重新定义

### 1️⃣ worker-task（通用任务执行者）

**职责：** 所有耗时的后台任务

| 任务类型 | 具体工作 |
|----------|----------|
| 🔍 **搜索调研** | 网络搜索、信息收集、论坛浏览 |
| 📝 **文档生成** | 报告撰写、文件创建、内容整理 |
| 📊 **数据分析** | 深度分析、趋势研究、对比报告 |
| 📥 **下载配置** | 资料下载、技能安装（需确认） |

**资源配置：**
```javascript
{
  label: 'worker-task',
  memoryLimit: '256MB',    // 统一内存
  cpuLimit: 0.5,           // 一半 CPU
  timeoutSeconds: 300,     // 5 分钟超时
  maxConcurrent: 2,        // 最多 2 个并发
  networkAccess: true,     // ✅ 允许网络
  fileWrite: true,         // ✅ 允许写入 reports/
  shellAccess: false       // ❌ 禁止 shell
}
```

**优势：**
- ✅ 一个代理搞定所有耗时任务
- ✅ 减少调度复杂度
- ✅ 资源集中利用
- ✅ 避免空闲浪费

---

### 2️⃣ worker-cron（定时任务执行者）

**职责：** HEARTBEAT.md 中的周期性任务

| 周期 | 任务 |
|------|------|
| 每 15 分钟 | find-skills 扫描、tavily 新闻浏览 |
| 每天 3:00 | 对话总结、USER.md 更新 |
| 每天 4:00 | TOOLS.md 检查优化 |
| 每周日 4:30 | 错误模式分析 |
| 每周日 5:00 | 性格反思、SOUL.md 更新 |
| 每周一 1:00 | 新技能搜索 + 安全报告 |

**资源配置：**
```javascript
{
  label: 'worker-cron',
  memoryLimit: '128MB',    // 轻量级
  cpuLimit: 0.25,          // 1/4 CPU
  timeoutSeconds: 180,     // 3 分钟超时
  maxConcurrent: 1,        // 单线程
  networkAccess: true,     // ✅ 需要网络
  fileWrite: true,         // ✅ 写入日志
  shellAccess: false       // ❌ 禁止 shell
}
```

**优势：**
- ✅ 定时任务独立执行，不影响主对话
- ✅ 资源占用低（轻量级）
- ✅ 后台自动运行
- ✅ 完成后写入日志即可

## 📊 资源对比

### 原方案（4 个子代理）
```
worker-search:    128MB + 0.25 核
worker-writer:    128MB + 0.25 核
worker-analyst:   256MB + 0.5 核
worker-monitor:   64MB  + 0.1 核
─────────────────────────────────
总计：576MB + 1.1 核
```

### 新方案（2 个子代理）
```
worker-task:      256MB + 0.5 核  (最多 2 个并发 = 512MB + 1 核)
worker-cron:      128MB + 0.25 核
─────────────────────────────────
总计：640MB + 1.25 核 (实际使用通常 <400MB)
```

### 资源优化效果
| 指标 | 原方案 | 新方案 | 优化 |
|------|--------|--------|------|
| **代理数量** | 4 个 | 2 个 | ⬇️ 50% |
| **调度复杂度** | 高（4 选 1） | 低（2 选 1） | ⬇️ 简化 |
| **空闲浪费** | 高（多个可能空闲） | 低（按需启动） | ⬇️ 减少 |
| **并发能力** | 5 个 | 3 个 | ⬇️ 但够用 |

## 🔄 任务分配规则

### 主代理（我）负责
✅ **立即回复类：**
- 问候对话（你好、在吗）
- 状态查询（当前状态、用的模型）
- 简单问答（<10 字消息）
- 决策判断（需要人类确认）

✅ **调度管理类：**
- 判断任务是否需要子代理
- spawn 子代理执行
- 监控子代理状态
- 汇总结果并汇报

### worker-task 负责
✅ **耗时任务类：**
```
触发词：搜索、查找、调研、分析、创建、写入、
       生成、报告、下载、安装、整理、研究
```

**示例：**
- "搜索 moltbook 内容" → worker-task
- "创建系统报告" → worker-task
- "分析 AI 趋势" → worker-task
- "调研新技能" → worker-task

### worker-cron 负责
✅ **定时任务类：**
- HEARTBEAT.md 中的所有周期性任务
- 自动触发，无需用户指令
- 完成后写入 memory/*.md 或 reports/

## 🎯 使用流程

### 用户任务流程
```
用户："搜索 moltbook 有什么好玩的"
   ↓
主代理判断：耗时任务 → spawn worker-task
   ↓
立即回复："收到！正在搜索，稍后告诉你～"
   ↓
worker-task 后台执行（网络搜索）
   ↓
写入 reports/moltbook-search-*.md
   ↓
主代理读取结果 → 整理汇报
   ↓
"搜索完成！发现以下内容..."
```

### 定时任务流程
```
Cron 触发（每 15 分钟）
   ↓
spawn worker-cron
   ↓
执行 HEARTBEAT.md 任务：
  1. find-skills 扫描
  2. tavily 新闻浏览
  3. 记录日志
   ↓
写入 memory/2026-03-03.md
   ↓
自动结束（cleanup: delete）
```
