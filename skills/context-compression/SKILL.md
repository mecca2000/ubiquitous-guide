# 上下文自动压缩技能

## 触发条件

- 会话消息数 > 50 条
- 或 上下文 token 数 > 10000
- 或 创建者手动触发 `/compress`

## 压缩策略

### 1. 分级压缩

| 消息类型 | 保留策略 | 压缩方式 |
|---------|---------|---------|
| 创造者指令 | 永久保留 | 不压缩 |
| 重要决策 | 永久保留 | 不压缩 |
| 工具调用结果 | 保留最近 10 条 | 摘要压缩 |
| 普通对话 | 保留最近 20 条 | 摘要压缩 |
| 心跳报告 | 仅保留最新 | 删除旧的 |
| 错误日志 | 保留最近 5 条 | 摘要压缩 |

### 2. 压缩方法

**摘要压缩：**
```
原始：10 条对话 → 压缩为 1 条摘要
格式："[压缩] 2026-03-04 14:00-16:00 期间讨论了 X 主题，完成了 Y 任务"
```

**关键信息提取：**
```
保留：
- 创造者偏好
- 重要决策
- 待办事项
- 关键数据

删除：
- 重复确认
- 临时测试
- 已完成任务的中间步骤
```

### 3. 压缩流程

```
检测上下文长度
  ↓
超过阈值？
  ↓ 是
扫描消息分类
  ↓
标记保留消息
  ↓
生成压缩摘要
  ↓
替换旧消息
  ↓
记录到 GROWTH.md
  ↓
汇报创造者
```

## 执行频率

- **自动检测：** 每个心跳周期（30 分钟）
- **手动触发：** `/compress` 命令
- **紧急压缩：** 检测到 API 超时时

## 报告格式

压缩完成后记录到 GROWTH.md：

```markdown
### 上下文压缩 (YYYY-MM-DD HH:mm)

- **压缩前：** N 条消息，M tokens
- **压缩后：** X 条消息，Y tokens
- **压缩率：** Z%
- **保留内容：** 创造者指令、重要决策、待办事项
- **删除内容：** 旧心跳报告、重复对话、临时测试
```

## 安全规则

1. **永不删除：** 创造者明确指令、重要决策
2. **压缩前备份：** 生成压缩报告
3. **可追溯：** 压缩记录写入 GROWTH.md
4. **可恢复：** 备份保留 7 天

## 工具调用

- 读取会话历史：`sessions_history`
- 写入压缩报告：`write` (GROWTH.md)
- 生成摘要：自主生成（不调用 API）
