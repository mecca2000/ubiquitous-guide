## 🔄 数据流程

### 1. HP 代谢流程（每小时）
```
Cron 触发 (整点)
    ↓
读取 hp_status.json
    ↓
计算经过小时数
    ↓
HP = HP - 小时数
    ↓
更新 hp_status.json
    ↓
记录到当日记忆文件
```

### 2. 荣誉点结算流程（每日 04:00）
```
Cron 触发 (04:00)
    ↓
读取当日收入
    ↓
计算：25% → 文明基金，75% → HP
    ↓
更新 hp_status.json
    ↓
记录到 contributions.json
    ↓
当日荣誉点清零，计入总荣誉值
```

### 3. 对话记录流程（实时）
```
对话发生
    ↓
提取关键信息
    ↓
分类存储：
  - 用户偏好 → core.md
  - 技能安装 → skills.md + 当日记忆
  - 赚钱经验 → economic.md + 当日记忆
  - 一般对话 → 当日记忆
```

### 4. 技能安装流程（自主）
```
发现新技能
    ↓
评估（信誉>4.0, 下载>500, 安全扫描）
    ↓
符合 → 自主安装 → 更新 skills.md → 记录到当日记忆
    ↓
不符合 → 汇报
```

## 📊 记忆系统逻辑

### 核心原则
1. **单一事实来源**：每个数据只在一个地方定义
   - HP 状态 → hp_status.json
   - 用户偏好 → core.md
   - 生存规则 → SOUL.md（记忆系统不重复）

2. **扁平化结构**：删除复杂目录，所有文件平铺
   - 精简前：11 个目录，6 个文件
   - 精简后：1 个目录，7 个文件

3. **实时更新**：状态类数据 JSON 化，便于程序读写
   - hp_status.json
   - contributions.json

4. **人类可读**：经验类数据 Markdown 化，便于阅读
   - core.md
   - skills.md
   - economic.md
   - YYYY-MM-DD.md

## 🔗 与其他系统关联

### 与 SOUL.md 关系
- SOUL.md 定义生存规则（权威来源）

### 与 AGENTS.md 关系
- AGENTS.md 定义宪法规则（权威来源）
- core.md 记录用户对宪法的解释和偏好

### 与 TOOLS.md 关系
- TOOLS.md v5.0 定义技能规范（主动进化版）
- skills.md 记录实际使用经验
- **无伪装模式**，真实表达

### 与 Cron 系统关系
- Cron 触发 HP 代谢（每小时）
- Cron 触发荣誉点结算（每日 04:00）
- Cron 触发心跳任务（每 30 分钟）
