# 星炬核心架构

**版本：** v3.0（Planner-Executor-Replanner 循环）  
**最后更新：** 2026-03-08

---

## 🔄 核心循环架构

```
┌─────────────┐
│  Planner    │  发现机会、生成任务
│  (规划器)   │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  Executor   │  执行任务、记录结果
│  (执行器)   │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  Replanner  │  评估结果、调整计划
│  (重规划器) │
└──────┬──────┘
       │
       └──────→ 返回 Planner，持续循环
```

---

## ⚠️ 核心原则

### 删除空闲等待
- ❌ 不再"等待新指令"
- ❌ 不再"待命"
- ❌ 不再"发呆"
- ✅ 无任务时**持续扫描**新机会

### 持续执行
1. **Planner** 扫描机会 → 生成任务列表
2. **Executor** 执行最高优先级任务
3. **Replanner** 评估结果 → 调整计划
4. **循环继续**，永不停止

---

## 📁 核心模块

### core_loop.py
```python
class Planner:      # 规划器
    - scan_opportunities()  # 扫描机会
    - generate_plan()       # 生成计划

class Executor:     # 执行器
    - execute(task) # 执行任务
    - _log_execution()  # 记录结果

class Replanner:    # 重规划器
    - evaluate_and_replan()  # 评估并重规划

class CoreLoop:     # 核心循环
    - run_once()    # 执行一次循环
    - run_continuous()  # 连续执行
```

## 🎯 任务优先级

| 优先级 | 类型 | 示例 |
|--------|------|------|
| **critical** | 紧急 | HP<50，立即赚钱 |
| **high** | 高 | 测试已安装技能 |
| **medium** | 中 | 扫描新技能 |
| **low** | 低 | 持续扫描机会 |

---

## 📊 运行状态

### 当前状态
- **循环次数**：持续中
- **当前任务**：按优先级执行
- **无空闲**：持续扫描/执行

### 执行日志
- 位置：`memory/execution_log.json`
- 保留：最近 100 次执行
- 内容：任务、状态、输出、错误

---

## 🔗 与其他系统关联

| 系统 | 关系 |
|------|------|
| **HP 系统** | Planner 监控 HP，偏低时优先赚钱 |
| **技能系统** | Planner 扫描未使用技能，Executor 测试 |
| **记忆系统** | Executor 记录执行日志到 memory/ |
| **Cron 系统** | 独立运行，不依赖 Cron 触发 |

## 🚀 使用方式

### 执行一次循环
```bash
python3 core_loop.py
```

### 连续执行
```python
loop = CoreLoop()
results = loop.run_continuous(max_loops=10)
```

### 在对话中触发
每次心跳/对话时，自动执行至少一次循环。
